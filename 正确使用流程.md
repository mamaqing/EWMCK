# 正确使用流程指南

## 浏览器环境限制说明

由于浏览器安全限制，我们无法直接在浏览器中执行文件上传到GitHub的操作。这就是为什么之前会出现`ReferenceError: require is not defined`错误的原因。

## 正确的使用流程

### 步骤1：修改分数

使用界面上的滑块修改各个维度的分数。

### 步骤2：生成报告图片

点击"生成并保存报告图片"按钮：
- 系统会自动生成最新的报告图片
- 图片会自动保存到您的电脑上
- 图片同时会保存在项目的`报告图片/`目录中

### 步骤3：上传图片到GitHub

打开命令行终端，执行以下命令：

```bash
# 重新构建项目并上传图片
npm run generate-and-upload

# 或者仅上传现有图片
npm run upload-report
```

### 步骤4：生成并扫描二维码

点击"生成二维码"按钮，然后扫描二维码查看最新的报告。

## 注意事项

1. **GitHub Pages更新延迟**：
   - 上传成功后，GitHub Pages通常需要1-5分钟才能更新
   - 建议上传后等待几分钟再扫描二维码

2. **图片文件大小**：
   - 我们已经优化了图片生成参数，平衡了质量和文件大小
   - 优化后图片文件大小将显著减小，加快上传和加载速度

3. **命令行操作**：
   - 确保您的电脑已安装Node.js和Git
   - 确保您已正确配置Git的GitHub凭证

## 故障排除

### 上传失败

如果上传失败，请检查：
1. 网络连接是否正常
2. Git配置是否正确
3. 您是否有GitHub仓库的推送权限

### 二维码显示旧报告

如果二维码显示旧报告：
1. 确保已执行上传命令
2. 等待GitHub Pages更新(通常需要几分钟)
3. 刷新页面并重新生成二维码

## 结论

虽然我们不能在浏览器中直接执行上传操作，但通过上述简单的步骤，您仍然可以轻松地将最新的报告图片上传到GitHub，并通过二维码访问。这个流程既安全又可靠，确保了您的代码和数据的安全性。
